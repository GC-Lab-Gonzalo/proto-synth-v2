02 · Hardware

El Proto-Synth v2

El pinout completo del Proto-Synth v2. Cada pin tiene una función específica.

  • MCUESP32 DevKit v1
  • Clock240 MHz dual-core
  • Audio DACGPIO 25 · 8-bit · 22kHz
  • ADC12-bit · 4 canales
  • Potenciómetros4 × (ADC2)
  • Botones4 × pull-up
  • LEDs4 × indicadores
  • IMUMPU6050 (I2C)
  • LDRGPIO 26
  • MicrófonoGPIO 33
  • MIDITX0 · 31250 baud
  • Librería audioMozzi / DAC nativo

⚠ ERRATA v2.0

Los potenciómetros están invertidos en el hardware v2.0. El firmware lo corrige automáticamente con 4095 - analogRead(PIN).

Antes de subir código, el POT3 debe estar al máximo (pin de strapping del ESP32).

03 · Setup

Instalación de Software

Seguí los pasos en orden. No hace falta experiencia previa — cada paso está explicado. Tildá cada uno cuando lo terminés.

1
Instalar Visual Studio Code (editor de código) ▼

Visual Studio Code (VSCode) es el editor de texto donde vas a escribir y ver el código. Es gratuito y es el más usado en el mundo. Aunque Arduino IDE también puede editar código, VSCode es mucho más cómodo para explorar archivos, hacer búsquedas y entender proyectos grandes.

Entrá al link, elegí tu sistema operativo (Windows / Mac / Linux) y descargá el instalador. Seguí los pasos de instalación como cualquier programa normal.

https://code.visualstudio.com/download

💡 Una vez instalado, abrí VSCode y explorá la carpeta del repositorio desde File → Open Folder.

2
Instalar Arduino IDE 2.x (para cargar código al ESP32) ▼

Arduino IDE es el programa que traduce el código a instrucciones que entiende el ESP32 y lo envía por el cable USB. Es gratuito. Descargá la versión 2.x — la versión 1.x está desactualizada y puede dar problemas con el ESP32.

Entrá al link y descargá el instalador para tu sistema operativo. La instalación es automática.

https://www.arduino.cc/en/software

💡 Al abrir Arduino IDE por primera vez va a aparecer en inglés — está bien así, lo usaremos en inglés.

3
Agregar soporte para ESP32 en Arduino IDE ▼

Arduino IDE viene preparado para placas Arduino (un tipo de microcontrolador). Para que también funcione con el ESP32 (que es el chip del Proto-Synth), hay que agregar un "paquete de soporte" extra. Solo se hace una vez.

Paso a paso:

1. Abrí Arduino IDE → menú File → Preferences

2. En el campo "Additional boards manager URLs", pegá esta URL exactamente:

https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json

3. Hacé click en OK para cerrar Preferences.

4. Menú Tools → Board → Boards Manager...

5. En el buscador escribí esp32 y buscá el paquete de Espressif Systems. Hacé click en Install. Puede tardar unos minutos.

💡 Si ves un error de descarga, verificá tu conexión a internet y volvé a intentar.

4
Seleccionar la placa correcta: ESP32 Dev Module ▼

Antes de cargar cualquier código, Arduino IDE necesita saber qué placa estás usando. Si no lo configurás, el código no va a funcionar.

Paso a paso:

1. Conectá el Proto-Synth al computador con el cable USB.

2. Menú Tools → Board → esp32 → buscá y seleccioná ESP32 Dev Module

3. Menú Tools → Port → seleccioná el puerto que aparece (en Windows: COM3, COM4, etc. En Mac: /dev/cu.usbserial...)

💡 Si no aparece ningún puerto, es posible que necesites instalar el driver del chip USB. En el troubleshooting de la sección anterior encontrás cómo.

5
Instalar librerías de audio y sensores ▼

Las librerías son paquetes de código que otros programadores ya hicieron, que podemos usar para no empezar de cero. Necesitamos tres:

Mozzi — genera sonido en el ESP32 (osciladores, filtros, síntesis)

Adafruit MPU6050 — lee el sensor de movimiento (IMU)

Adafruit Unified Sensor — requerida por la librería del IMU

Para instalar cada una: Tools → Manage Libraries... → buscá el nombre exacto → Install

Mozzi
Adafruit MPU6050
Adafruit Unified Sensor

💡 Si Arduino IDE te pregunta si instalás las dependencias, elegí "Install All".

6
Descargar el repositorio del Proto-Synth ▼

El repositorio es la carpeta en GitHub que tiene todos los firmwares y el código del proyecto. Lo podés descargar de dos formas:

Opción A (recomendada, sin instalar nada extra): Entrá al link → hacé click en el botón verde Code → elegí Download ZIP → descomprimí la carpeta en un lugar fácil de encontrar (por ejemplo, tu escritorio o Documentos).

Opción B (con Git):

git clone https://github.com/GC-Lab-Gonzalo/proto-synth-v2
https://github.com/GC-Lab-Gonzalo/proto-synth-v2

💡 Una vez descargado, abrí la carpeta en VSCode para explorar todos los firmwares disponibles.

// TROUBLESHOOTING COMÚN

No aparece el puerto COM en Arduino IDE

Instalá el driver del chip USB-serial del ESP32. Para CH340: buscar "CH340 driver". Para CP2102: buscar "CP2102 driver Silicon Labs". Desconectá y reconectá el USB después de instalar.

Error al compilar: "too many arguments to timerBegin"

Estás usando el ESP32 Arduino Core v3.x. La API de timers cambió. Los firmwares del repo ya son compatibles. Si tenés código propio, usá: timerBegin(freq_hz) en lugar de timerBegin(0, 80, true).

El upload falla (error de comunicación)

Verificá que el POT3 esté al máximo (es un pin de strapping del ESP32 y debe estar HIGH durante el boot). También probá mantener presionado el botón BOOT del ESP32 mientras hacés click en Upload.

Los potenciómetros funcionan al revés

Es una errata conocida del hardware v2.0. El firmware lo corrige con 4095 - analogRead(PIN). Si estás escribiendo código propio, recordá aplicar esta inversión.

No hay sonido en la salida de audio

El DAC del ESP32 no puede manejar auriculares directamente — necesitás un amplificador o un adaptador activo. También verificá que el firmware esté corriendo (los LEDs deben estar activos) y que el jack esté completamente insertado.
